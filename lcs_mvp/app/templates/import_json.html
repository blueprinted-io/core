Generate content with an external LLM

Copy a prompt, paste it into Claude, ChatGPT, or any capable LLM, then append your source documentation. The output will be valid import JSON.

Task extraction

Extracts structured tasks with title, outcome, steps (including CLI commands and completion criteria), facts, concepts, and dependencies from procedural documentation.

Primer extraction

Extracts conceptual primers with rich markdown explanation covering how something works, trade-offs, and when to choose each approach. For explanatory or reference documentation, not procedural content.

JSON shape reference

The top-level object may contain tasks, primers, and/or workflows arrays. Required fields are marked.

Task 
{
  "title": "...",            // required — noun phrase, 5-10 words, no leading verb
  "outcome": "...",          // required — passive voice, one sentence
  "procedure_name": "...",   // short imperative phrase; defaults to title if omitted
  "software_name": "...",    // or null
  "software_version": "...", // or null
  "facts": [                 // what the components are; [] if none
    "Sentence describing a component or concept."
  ],
  "concepts": [              // why this specific task must be done; [] if none
    "Paragraph explaining the consequence of skipping this task."
  ],
  "dependencies": [          // preconditions; [] if none
    "Machine is accessible with sudo privileges."
  ],
  "irreversible": false,     // true if hard to undo (e.g. disk format, data deletion)
  "steps": [
    {
      "text": "...",         // required — concrete verb, one action only
      "completion": "...",   // required — specific observable confirmation
      "actions": [           // CLI commands, menu paths, shortcuts; [] if self-explanatory
        "sudo apt install example"
      ],
      "notes": null          // edge cases or caveats from source; null if none
    }
  ],
  "record_id": "...",        // UUID; auto-generated if omitted
  "version": 1               // integer; defaults to 1
}
Primer 
{
  "title": "...",            // required — noun phrase, 5-12 words
  "summary": "...",          // required — one sentence, active voice
  "explanation": "...",      // required — full conceptual content in markdown
  "analogies": null,         // string or null
  "software_name": "...",    // or null
  "software_version": "...", // or null
  "domain": "",              // optional domain slug
  "record_id": "...",        // UUID; auto-generated if omitted
  "version": 1               // integer; defaults to 1
}
Workflow 
{
  "title": "...",        // required
  "objective": "...",    // required — one sentence describing the operational goal
  "task_refs": [         // required — ordered confirmed tasks; at least one
    {"record_id": "...", "version": 1}
  ],
  "record_id": "...",    // UUID; auto-generated if omitted
  "version": 1           // integer; defaults to 1
}
